## Metodología de Estimación – Planning Poker

Para la estimación de las historias de usuario se utilizó la técnica **Planning Poker**, una práctica ágil de Scrum basada en consenso.

Se empleó la serie de Fibonacci para asignar los Story Points:

| Serie Fibonacci Utilizada |
|----------------------------|
| 1 |
| 2 |
| 3 |
| 5 |
| 8 |
| 13 |

Los Story Points representan una medida relativa del esfuerzo y complejidad, no tiempo exacto en horas.

---

## Criterios de Evaluación para la Estimación

Para determinar los Story Points de cada historia se evaluaron los siguientes factores:

| Criterio | Descripción |
|----------|-------------|
| Complejidad Técnica | Nivel de dificultad en el desarrollo |
| Riesgo | Posibles problemas técnicos o funcionales |
| Esfuerzo | Cantidad de trabajo requerido |
| Dependencias | Relación con otros módulos |
| Incertidumbre | Nivel de claridad en los requisitos |

## Justificación

La asignación de Story Points se realizó considerando la complejidad relativa entre historias. Las funcionalidades que implican validaciones, cálculos o integración entre módulos recibieron mayor puntuación debido a su mayor esfuerzo e incertidumbre.

Aunque el proyecto es individual, se aplicó un análisis estructurado siguiendo la metodología Scrum.

Se utilizaron dos métodos de priorización:

1. Valor de negocio (Alto, Medio, Bajo)
2. Estimación mediante Story Points usando la serie de Fibonacci (1, 2, 3, 5, 8, 13).
