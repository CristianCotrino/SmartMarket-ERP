# Definition of Done (DoD)

La Definition of Done establece los criterios que deben cumplirse para considerar una historia de usuario como finalizada dentro del Sprint.

Una historia de usuario se considera terminada cuando cumple con los siguientes puntos:

## Criterios Técnicos

- El código ha sido implementado correctamente según los requisitos establecidos.
- La funcionalidad cumple con los criterios de aceptación definidos en la historia de usuario.
- La base de datos ha sido actualizada y validada correctamente.
- No existen errores críticos que afecten la funcionalidad principal.

## Criterios de Calidad

- Se han ejecutado pruebas unitarias y pruebas funcionales.
- Los casos de prueba definidos en el Test Plan han sido verificados.
- No se presentan fallos en los escenarios principales de uso.
- Se valida el correcto manejo de datos inválidos o excepciones.

## Criterios de Integración

- La funcionalidad desarrollada no afecta negativamente otros módulos del sistema.
- Se verificó la correcta integración con los módulos relacionados (Inventario, Ventas, Compras, etc.).

## Criterios de Documentación

- El código ha sido subido correctamente al repositorio GitHub.
- La documentación correspondiente ha sido actualizada.
- Se registran los cambios realizados durante el Sprint.

## Aprobación Final

- La funcionalidad es revisada y validada.
- Cumple con el objetivo definido en el Sprint Planning.

---

La Definition of Done garantiza que cada incremento del sistema entregue valor real y cumpla estándares mínimos de calidad antes de considerarse finalizado.
